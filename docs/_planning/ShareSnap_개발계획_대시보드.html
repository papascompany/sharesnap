User Flow
핵심 사용자 흐름
모임장이 공유방을 만들고, 참여자들이 사진을 공유하고, 포토북을 만들어 주문하기까지의 전체 여정
01
🏠

공유방 개설

모임장이 여행/모임명으로 공유방을 만들고, 카카오톡으로 초대 링크를 공유합니다.

02
🔗

카카오 로그인 참여

초대 링크 클릭 → 카카오 로그인(원클릭) → 공유방 자동 입장. 별도 회원가입 없이 바로 시작합니다.

03
📸

사진 업로드 & 채팅

여행 중 PWA 앱에서 사진 촬영/업로드. 실시간 채팅으로 코멘트를 남기며 추억을 공유합니다.

04
🖼️

갤러리에서 모아보기

모인 사진을 타임라인/그리드 뷰로 감상. 포토북에 넣을 사진을 선택합니다.

05
✨

포토북 자동편집

선택한 사진+코멘트를 1페이지 1사진으로 자동 배치. Fabric.js 편집기로 표지/내지를 커스텀합니다.

06
📦

PDF 생성 & 주문

300dpi 고해상도 PDF를 생성하고, 미리보기 후 포토북/사진 인화를 주문합니다.

Development Phases
7 Phase 개발 로드맵
Foundation부터 Deploy까지 단계적으로 진행. 각 Phase는 완료 조건을 모두 충족해야 다음 Phase로 전환됩니다.
Phase 1

Foundation

2~3일
프로젝트 셋업, Supabase 연결, 인증(카카오 로그인), 전체 DB 스키마 마이그레이션
  • Next.js 14 프로젝트 초기화 (App Router + TypeScript + Tailwind)
  • shadcn/ui + Zustand + Supabase 패키지 설치
  • 모듈 디렉토리 구조 생성 (M0~M9)
  • Supabase 클라이언트 설정 (Client/Server/Middleware)
  • 공용 모듈(M0): Layout, ErrorBoundary, 타입 정의, DPI 유틸
  • DB 마이그레이션: 9개 테이블 + RLS 정책 + Storage 버킷
  • 인증 모듈(M1): 카카오 로그인, AuthGuard, 로그인 페이지
완료 조건: npm run build 성공, 카카오 로그인/로그아웃 동작, DB 스키마 완료
Phase 2

Room & Chat

3~4일
공유방 생성/참여, 실시간 채팅, 카카오톡 공유 초대
  • 공유방 모듈(M2): 방 CRUD, share_code 기반 초대 링크, 멤버 관리
  • 채팅 모듈(M3): Supabase Realtime, 메시지 송수신, Presence 표시
  • 카카오톡 공유 연동: Kakao JS SDK, Feed 메시지로 초대
  • 모바일 반응형 채팅 UI, 하단 입력창 고정
완료 조건: 방 생성/참여, 실시간 채팅, 카카오톡 초대 링크 공유 동작
Phase 3

Photo Management

3~4일
사진 업로드, 이미지 처리 파이프라인, 갤러리, 코멘트
  • 다중 사진 업로드 (드래그&드롭 + 모바일 카메라)
  • 이미지 처리: 원본 저장, 썸네일(400px), 중간(1200px) 자동 생성
  • EXIF 추출(촬영일시, GPS), 방향 자동 보정
  • 갤러리 뷰: 그리드(3/4열) + 타임라인(날짜별) + 전체화면 뷰어
  • 사진 코멘트, 포토북/인화용 사진 선택 토글
완료 조건: 다중 업로드, 썸네일 생성, 갤러리 뷰 전환, 코멘트 작성
Phase 4 ⭐

Fabric.js Editor

5~7일
Fabric.js 기반 표지/내지 편집기. 모바일 터치 최적화 필수. 프로젝트의 핵심 모듈.
  • Fabric.js 캔버스: dynamic import(ssr:false), 사이즈별 300dpi 초기화
  • 텍스트 도구: 서버 폰트 로드, 크기/색상/정렬
  • 이미지 배치: 사진 추가/교체, 크롭/리사이즈, 필터
  • 도형, 클립아트(서버 제공), 배경(단색/그라데이션/이미지)
  • 모바일 최적화: 핀치 줌/팬, 터치 선택/이동, 하단 도구 모음
  • 히스토리: Undo/Redo, JSON 저장/불러오기, 자동 저장
  • 편집 리소스 로드: 폰트/클립아트/배경 API 연동
완료 조건: SSR 에러 없음, 편집 도구 전체 동작, 모바일 터치, Undo/Redo, JSON 직렬화
Phase 5

Photobook + PDF

5~7일
포토북 자동편집, 표지/내지 커스텀, 300dpi PDF 서버사이드 생성
  • 포토북 생성 위저드: 사이즈 선택 → 사진 선택(순서 드래그) → 코멘트 매칭
  • 자동 레이아웃 엔진: 1페이지 1사진, 사진비율 최적화, Fabric.js JSON 자동 생성
  • 표지 편집: 앞/뒷표지 기본 템플릿 → 편집기(M5) 연결
  • 내지 커스텀: 자동편집 결과 미리보기, 개별 페이지 편집기 연결
  • 전체 미리보기: 플립북/썸네일 스타일
  • PDF 생성(서버): Fabric.js JSON → Puppeteer 렌더링 → 300dpi PDF → 병합
완료 조건: 자동 레이아웃, 편집기 연동, 300dpi PDF 다운로드
Phase 6

Orders & Admin

3~4일
인화 주문, 포토북 주문, 관리자 리소스 관리
  • 인화 주문: 사진 선택 → 사이즈/수량 → 가격 계산 → 배송지 → 주문
  • 포토북 주문: PDF 생성 완료 → 가격(사이즈×페이지수) → 주문
  • 관리자: 폰트/클립아트/배경이미지 업로드 관리 (CRUD)
  • 주문 대시보드, role-based 관리자 인증
완료 조건: 전체 주문 플로우, 리소스 관리 CRUD
Phase 7

Integration & Deploy

2~3일
전체 통합 테스트, 성능 최적화, Vercel 프로덕션 배포
  • E2E 테스트: 회원가입 → 방 생성 → 사진 업로드 → 포토북 → PDF → 주문
  • 성능 최적화: 이미지 lazy loading, PDF 생성 큐, 모바일 프로파일링
  • Vercel 배포, 도메인/SSL, 모니터링 설정
완료 조건: E2E 통과, Vercel 프로덕션 배포 완료
